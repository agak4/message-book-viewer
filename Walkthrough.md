# Walkthrough

## 목표 (Objective)
- 북마크나 프로그레스 바를 통해 여러 페이지를 이동할 때 발생하는 촤라락 애니메이션 속도를 총합 1초 이내로 단축.
- 잦은 조작 시 `setTimeout`이 겹치거나 취소되지 않는 문제(클린업)가 발생하지 않도록 로직의 견고함 확보.

## 작업 내역 (Changes Made)
1. **동적 애니메이션 속도 제어 로직 추가 (`script.js`)**
   - 한번에 넘어가는 페이지 수(`totalFlips`)가 많아질수록 개별 페이지 회전 속도와 간격을 자동으로 줄여, 어떤 거리든 **최대 1초 이내**에 도착하도록 구현했습니다.

2. **`setTimeout` 초기화 및 메모리 관리 보강 (`script.js`)**
   - `applyPageFlip` 함수 호출 시, 해당 페이지에 예약된 이전 타이머(`delay`, `duration`)가 있다면 `forEach(clearTimeout)`를 통해 즉시 취소하도록 확정했습니다.
   - 애니메이션이 모두 끝난 후에는 `pageTimeouts` 맵에서 해당 인덱스를 삭제하여 불필요한 메모리 사용을 방지하도록 개선했습니다.
   - 빠른 연속 클릭 시에도 이전 애니메이션의 잔상이 남지 않도록 엔진 리플로우(`p.offsetWidth`)를 강제하고 스타일을 명확히 리셋합니다.

## 테스트 및 검증 방향 (Verification Plan)
- 프로그레스 바를 매우 빠르게 좌우로 흔들거나 북마크를 연달아 클릭했을 때, 페이지가 엉키거나 특정 z-index에 멈춰있는 현상이 없는지 확인.
- 많은 양의 페이지가 넘어갈 때 1초 이내에 촤라락 감기며 부드럽게 마무리되는지 확인.
- 개발자 도구의 성능 탭이나 메모리 점검 시 타이머가 무한정 증식하지 않는지 확인.
