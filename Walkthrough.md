# Walkthrough

## 목표 (Objective)
- 북마크나 프로그레스 바를 통해 여러 페이지를 이동할 때 발생하는 촤라락 애니메이션 속도를 총합 1초 이내로 대폭 단축.
- (관련 해결된 이전 이슈) 페이지 애니메이션 이동 중 z-index 충돌 문제 수정 및 앞/뒤 동일한 순차 넘김 효과 구현 완료.

## 작업 내역 (Changes Made)
1. **동적 애니메이션 속도 제어 로직 추가 (`script.js`)**
   - 한번에 넘어가는 페이지 수(`totalFlips`)가 2장 이상일 경우, 각 페이지가 넘어가는 시간(`duration`)과 페이지 사이의 딜레이(`staggerDelay`)를 동적으로 계산하도록 로직을 수정했습니다.
   - 전체 애니메이션이 시작하여 모두 완전히 끝나는 총 소요 시간이 정확히 `1000ms`(1초) 이내가 되도록 비례식을 적용하였습니다. 
   - 페이지 장수가 많아질수록 딜레이와 개별 페이지 회전 속도가 자연스럽게 더 빨라져 시각적으로 부드럽고 쾌적한 피드백을 유지합니다. 
   - 한 장만 넘길 때는 본래 설정되었던 종이 넘기는 템포(1.2초)가 적용될 수 있도록 일관성 있게 구현했습니다.

## 테스트 및 검증 방향 (Verification Plan)
- 프로그레스 바 조작이나 북마크를 눌러 맨 앞장 ↔ 맨 뒷장 등 많은 양의 페이지를 넘길 때 체감 속도가 이전에 비해 현저히 빨라졌는지, 최대 1초 내에 모든 페이지가 모두 뒤집히는지 확인.
- 1장씩 앞/뒤로 책을 넘길 때는 기존과 동일한 여유로운 속도로 잘 넘어가는지 확인.
- 페이지 전환 속도가 가변적으로 빨라진 도중 방향을 꺾었을 때(급하게 반복클릭)도 글리치가 없는지 확인.
