# Walkthrough - 애니메이션 OFF 시 이전 페이지 점멸 현상 수정

## 목표 (Objective)
- 사이드 컨트롤 패널에서 플립 애니메이션을 OFF 했을 때, 이전 페이지로 넘기는 동작에서 우측 페이지가 점멸하는 버그 픽스 (기존 기능 영향도 없음).

## 반영 내역 (Changes Made)
1. **페이지 전환 즉시 처리 적용** (`script.js`)
   - `applyPageFlip()` 함수 로직 최상단에, `duration`과 `delay`가 0일 경우 기존의 `setTimeout`을 통한 스케줄링(addTimer) 흐름을 타지 않고 즉시 목표 상태의 클래스 토글과 `zIndex` 업데이트가 이루어지도록 우회(bypass) 코드를 추가했습니다.
   - 이를 통해 브라우저 이벤트 루프에 의한 수 밀리초(ms) 지연 타이밍에 페이크 z-index(`zIndexDuringFlip`) 레이어가 화면에 노출됨으로써 눈에 띄던 깜빡임(Flicker) 현상을 근본적으로 차단했습니다.

## 테스트 및 검증 방향 (Verification Plan)
- 컨트롤 패널을 열어 애니메이션 버튼을 껐을 때(off).
- 키보드 왼쪽 방향키, 이전 페이지(◀) 버튼을 누르거나 프로그레스바 또는 북마크로 뒤로 건너뛰기를 할 때, 우측 페이지가 번쩍이는 점멸 현상 없이 깔끔하게 전환되는지 육안 점검.
